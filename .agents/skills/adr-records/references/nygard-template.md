# Nygard ADR template

Source: [Documenting Architecture Decisions](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions) · [adr.github.io](https://adr.github.io/)

Minimal format; valid MADR subset. Use when the repo already uses this shape.

```markdown
# {Number}. {Title}

Date: {YYYY-MM-DD}

## Status

{Proposed | Accepted | Deprecated | Superseded by [ADR-NNNN](NNNN-title.md)}

## Context

{Forces at play: technical, political, social, project-local.}

## Decision

{What we will do — active voice, full sentences.}

## Consequences

{Positive, negative, and neutral outcomes of applying this decision.}
```

Filename: match repo convention (`0001-title.md` or `NNNN-title.md`).
