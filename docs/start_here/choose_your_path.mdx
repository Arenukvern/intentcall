## Library, CLI, Server, and Adapter Paths

IntentCall is the direct platform layer when you are building a library, CLI, server, browser adapter, or platform bridge. The safe pattern is:

1. Keep business logic in Dart handlers registered with `AgentRegistry`.
2. Let adapters publish metadata and translate transport calls.
3. Invoke by the stored registry key from `AgentRegistry.listEntries()`.
4. Keep runtime discovery, product policy, and UI-specific behavior in the host app or tool.
5. Use `intentcall_cli` for framework-neutral manifest export and platform sync.

Do **not** depend on deleted packages `intentcall_apple` / `intentcall_android`. Runtime = `intentcall_platform`; projection = `intentcall_platform_sync`.

Flutter app authors who want the packaged product harness should start with [mcp_flutter](https://github.com/Arenukvern/mcp_flutter). Audience map: [Who is this for?](/start_here/audiences).
