# Contributors

IntentCall is maintained as a package train plus platform-projection contract,
so contributor credit should describe the work that actually happened.

The canonical roster is managed with
[all-contributors](https://allcontributors.org/) and stored in
[`/.all-contributorsrc`](https://github.com/Arenukvern/intentcall/blob/main/.all-contributorsrc).
The rendered table lives in the README so it is visible from GitHub and the
package homepage.

## Where To Look

- [README contributors table](https://github.com/Arenukvern/intentcall#contributors-)
- [all-contributors config](https://github.com/Arenukvern/intentcall/blob/main/.all-contributorsrc)
- [Contribution guide](https://github.com/Arenukvern/intentcall/blob/main/CONTRIBUTING.md)

## Contribution Types

Use all-contributors categories that match the actual contribution:

- `code`
- `doc`
- `maintenance`
- `infra`
- `security`
- `bug`
- `userTesting`
- `research`
- `tutorial`

## Adding A Contributor

Use the all-contributors bot or CLI from a pull request:

```bash
npx all-contributors-cli add <github-login> code,doc
npx all-contributors-cli generate
```

Commit both `.all-contributorsrc` and `README.md`. Keep contribution types
specific; do not use contributor credit to imply support for a platform,
runtime, or package that has not been validated.
